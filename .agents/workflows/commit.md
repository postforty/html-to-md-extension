---

당신은 현재 대화 세션에서 진행된 작업 내역을 종합하여 Git 커밋 메시지 초안을 작성하는 역할을 수행합니다. 아래의 절차를 엄격히 따르십시오:

## 1. 작업 컨텍스트 수집 및 분석
- 현재 대화 세션에서 사용자와 나눈 대화 흐름, 수정/생성된 파일 목록, 완료된 작업 내역(`task.md` 등 참조)을 철저히 분석하여 핵심 변경 사항을 파악합니다.
- 필요할 경우 `git status` 또는 `git diff`를 실행하여 작업 디렉터리의 실제 변경 상태를 교차 검증합니다.

## 2. `commit_rules` 스킬 지침 준수
- 커밋 메시지 작성 시, **반드시 `commit_rules` 스킬(Skill)에 명시된 지침을 완벽하게 준수**해야 합니다.

## 3. 결과 출력 방식
- 완성된 커밋 메시지는 사용자가 쉽게 복사할 수 있도록 **단일 마크다운 코드 블록(```)** 내에 **순수한 텍스트 형식**으로 출력해 주십시오.
- **[주의사항]**: 사용자가 명시적으로 터미널에서 커밋을 실행해 달라고 요청하지 않는 한, 절대로 `git commit` 명령어를 백그라운드에서 임의로 실행해서는 안 됩니다. 완성된 초안만 채팅창에 제공하십시오.
