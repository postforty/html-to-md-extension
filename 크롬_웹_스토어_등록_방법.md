개발하신 "HTML to MD Saver" 확장 프로그램을 전 세계 사용자들이 사용할 수 있도록 **크롬 웹 스토어(Chrome Web Store)에 등록하는 전체 과정**을 단계별로 안내해 드립니다.

---

### 1단계: 구글 개발자 계정 등록 (최초 1회)
크롬 웹 스토어에 앱을 올리려면 구글 개발자 계정이 필요합니다.
1. [Chrome 웹 스토어 개발자 대시보드](https://chrome.google.com/webstore/devconsole/)에 접속하여 구글 계정으로 로그인합니다.
2. 개발자 계약에 동의합니다.
3. **등록비 결제**: 최초 1회에 한해 **$5.00 (미화 5달러)**의 등록 수수료를 결제해야 합니다. (이후 평생 무료)

### 2단계: 프로젝트 압축하기 (.zip)
업로드를 위해 소스 코드를 하나의 ZIP 파일로 압축해야 합니다. 이때 불필요한 파일은 제외하는 것이 좋습니다.

*   **포함해야 할 파일:** `manifest.json`, `content.js`, `background.js`, `popup.html`, `popup.js`, `options.html`, `options.js`, 각종 라이브러리(`turndown.js` 등), 아이콘 이미지(`icon.png`)
*   **제외할 파일 (선택사항):** `.git` 폴더, `.gitignore`, `.env`, `README.md`, `idea.md`, `test.html` 등의 개발/테스트용 파일이나 민감한 정보가 담긴 파일. (특히 `.env`에 API 키가 있다면 반드시 제외하고 서버를 통하거나 사용자에게 직접 입력받도록 수정해야 합니다.)

해당 파일들만 선택하여 `html-to-md-saver.zip` 과 같은 이름으로 압축합니다. **주의할 점은 최상위 폴더 자체를 압축하는 것이 아니라, 폴더 내부의 파일들을 전체 선택해서 압축해야 `manifest.json`이 압축 파일의 최상단 경로에 위치하게 됩니다.**

### 3단계: 대시보드에 업로드
1. 다시 [개발자 대시보드](https://chrome.google.com/webstore/devconsole/)로 이동합니다.
2. 우측 상단의 **[+ 새 항목 (New Item)]** 버튼을 클릭합니다.
3. 만들어둔 `.zip` 파일을 드래그 앤 드롭하거나 선택하여 업로드합니다.

### 4단계: 스토어 등록 정보(Store Listing) 작성
업로드가 완료되면 상점 페이지에 노출될 정보를 입력해야 합니다.

1.  **기본 정보:**
    *   **이름, 요약, 설명**: `manifest.json`에 있는 내용이 기본으로 들어가지만, 스토어 노출을 위해 상세하게 작성합니다. (이 확장 프로그램의 장점인 다중 선택, GFM 지원, 코드 블록 보존 등을 강조하면 좋습니다.)
    *   **카테고리**: '생산성(Productivity)' 또는 '개발자 도구(Developer Tools)'가 적합합니다.
2.  **그래픽 자산 (매우 중요):**
    *   **스토어 아이콘**: `128x128` 픽셀 이미지
    *   **스크린샷**: 최소 1개, 최대 5개 (`1280x800` 또는 `640x400` 픽셀). 확장 프로그램을 실제로 사용하는 모습(하이라이트 되는 모습, 다운로드된 마크다운 결과 등)을 보여주어야 합니다.
    *   **프로모션 이미지 (선택):** `440x280` 픽셀
3.  **개인정보 처리방침 (Privacy Policy):**
    *   사용자의 데이터를 수집하지 않더라도, 최소한의 형식적인 개인정보 처리방침 URL이 필요할 수 있습니다. (GitHub Pages나 Notion 공유 링크로 간단히 작성하여 링크해도 됩니다.)

### 5단계: 권한 검토 (개인정보 보호 탭)
이 단계가 심사 통과에 가장 중요합니다. 크롬 웹 스토어는 권한(`permissions`) 요구에 매우 엄격합니다.

*   `manifest.json`에 선언된 권한(`activeTab`, `scripting`, `downloads`, `storage`)에 대해 **단일 목적(Single Purpose)** 원칙에 따라 "이 권한이 왜 꼭 필요한지" 구체적으로 서술해야 합니다.
    *   *예: `activeTab`, `scripting`: 현재 사용자가 보고 있는 웹 페이지에 스크립트를 주입하여 영역을 선택하고 HTML을 읽어오기 위해 필요합니다.*
    *   *예: `downloads`: 변환된 마크다운 데이터를 사용자 PC에 .md 파일로 저장하기 위해 필요합니다.*

### 6단계: 제출 및 심사 (Review)
모든 필수 항목을 채웠다면 우측 상단의 **[검토를 위해 제출(Submit for Review)]** 버튼을 클릭합니다.

*   **심사 기간**: 빠르면 몇 시간, 권한이 복잡하거나 주말이 끼어있으면 최대 2~3일 정도 소요될 수 있습니다.
*   심사가 통과되면 대시보드 상태가 '게시됨(Published)'으로 변경되며, 즉시 크롬 웹 스토어에서 검색하고 설치할 수 있게 됩니다.

---

**💡 배포 전 꼭 체크할 꿀팁:**
*   만약 코드 내부(예: `background.js`)에 **Gemini API Key 등 민감한 인증 정보가 하드코딩 되어 있다면 절대 그대로 올리시면 안 됩니다.** 스토어 심사에서 반려되거나 키가 탈취될 수 있습니다. (사용자가 `options.html`에서 자신의 키를 입력하게 하거나 별도 서버를 구축해야 합니다.)
*   배포 후 업데이트를 할 때는 코드를 수정한 뒤 `manifest.json`의 `"version"`을 `1.1` 등으로 올리고 다시 압축하여 대시보드에서 **패키지 업데이트**를 진행하시면 됩니다.
